# 语音绘图工具 — 设计文档

> 版本：1.0 | 日期：2026-06-12 | 状态：核心功能已完成

---

## 1. 项目定位

纯本地中文语音指令驱动 SVG 画布绘图工具。**零 AI 依赖**，基于浏览器 Web Speech API 实现语音→指令→图形的完整链路。

### 设计原则

- **零网络依赖**：全部解析在本地完成，不调用任何外部 API
- **容错优先**：语音识别同音字错误自动纠正
- **模块可拆分**：每个模块职责单一，可独立替换
- **渐进增强**：语音不可用时自动降级为文字输入

## 3. 同音字容错系统 — ✅ 已实现

语音识别引擎对中文同音字的错误率较高，系统内置 **150+ 条容错规则**，覆盖 5 大类别：

### 3.1 容错规则分布

| 类别 | 容错词数量 | 典型示例 |
|------|-----------|---------|
| 图形 | 60+ | 元→圆、巨型→矩形、五角行→五角星、蚊子→文字、妥圆→椭圆 |
| 颜色 | 50+ | 虹→红、兰→蓝、皇→黄、子→紫、百→白 |
| 尺寸 | 13 | 打→大、晓→小、钟→中 |
| 方位 | 20+ | 作→左、又→右、商→上、夏→下、中坚→中间 |
| 动作 | 40+ | 话→画、彻消→撤销、情空→清空、闪烁→删除、编大→变大 |

### 3.2 匹配策略

- **长词优先**：避免短词误匹配（如"蓝色"在"蓝"之前检查，防止只匹配到"蓝"）
- **类别隔离**：图形关键词只匹配图形，不会错误触发颜色匹配
- **不修改输入**：保持原始语音文本用于 UI 展示

## 4. 模块架构

```
用户语音/文字输入
        │
        ▼
┌─────────────────────────┐
│  useSpeechRecognition   │  Web Speech API 封装
│  (语音→文字)            │
└──────────┬──────────────┘
           │ raw text
           ▼
┌─────────────────────────┐
│  parser.ts              │
│  normalizeInput()       │  同音字归一化（可选，当前未启用全局归一化）
│  splitCommand()         │  复合指令拆分
│  parseOneCommand() →    │  单指令解析
│     parseShape()        │    图形模糊匹配 → SHAPE_KEYWORDS
│     parseColor()        │    颜色模糊匹配 → COLOR_KEYWORDS
│     parseSize()         │    尺寸模糊匹配 → SIZE_KEYWORDS
│     parsePosition()     │    位置模糊匹配 → POS_KEYWORDS
│  → Command[]            │
└──────────┬──────────────┘
           │ Command[]
           ▼
┌─────────────────────────┐
│  engine.ts              │
│  applyCommand()         │  指令执行
│    draw  → createDrawable() │
│    move  → findTarget() + 坐标更新 │
│    recolor → findTarget() + 颜色更新 │
│    resize → findTarget() + 缩放因子 │
│    delete → findTarget() + 过滤 │
│    undo  → 数组 pop     │
│    clear → 数组清空     │
│  → Drawable[]           │
└──────────┬──────────────┘
           │ Drawable[]
           ▼
┌─────────────────────────┐
│  main.tsx               │
│  renderShape()          │  SVG 渲染 13 种图形
│    circle  → <circle>   │
│    rect    → <rect>     │
│    triangle → <polygon> │
│    ellipse → <ellipse>  │
│    star    → <polygon>  │
│    line    → <line>     │
│    text    → <text>     │
│    arrow   → <g> + <polygon> │
│    diamond → <polygon>  │
│    heart   → <path>     │
│    arc     → <path>     │
│    hexagon → <polygon>  │
│    cloud   → <path>     │
└─────────────────────────┘
```

## 5. 未完成能力及原因

### 5.1 略过不做（有明确原因）

| 能力 | 原因 |
|------|------|
| 自定义像素尺寸 | 语音识别数字（如"100"→"一百"/"1百"）准确率极低，实现后用户体验差 |
| 自定义坐标位置 | 同上，数字坐标语音识别不可靠 |
| "以线为半径画圆" | 需要跟踪图形间引用关系，大幅增加引擎复杂度，实际使用场景少 |
| "从上一个图形延伸" | 需要状态记忆和上下文理解，超出关键词匹配范畴 |
| 第 4 个及以后的序号 | 画布上限约 20 个图形，定位第 4+ 的需求极少 |
| 多轮对话纠错 | 需要对话状态管理，属于 NLP 范畴而非关键词匹配 |

### 5.2 待规划（合理但暂未实现）

| 能力 | 优先级 | 说明 |
|------|--------|------|
| 旋转图形 | 中 | SVG transform 可实现，但语音描述角度困难（"旋转45度"→数字问题） |
| 图层管理（置顶/置底） | 中 | 数组重排即可，交互 UI 需要额外设计 |
| 画布导出 PNG/SVG | 高 | 纯前端实现，技术成熟，待排期 |
| 拖拽交互 | 中 | 需要接入鼠标/触摸事件，与语音并行 |
| 图形编组 | 低 | "把这三个图形编成一组"需要选择逻辑 |
| 复制/粘贴 | 中 | "复制这个圆然后移到右边" 待实现 |
| 线条方向（水平/垂直/斜线） | 高 | "画一条水平线"/"画一条斜线"，扩展 line 绘制逻辑 |
| 填充/描边切换 | 低 | 仅轮廓或仅填充模式 |
| 语音接入国内服务 | 中 | 百度/讯飞 API 可解决 Chrome Web Speech 国内不可用问题 |

### 5.3 会做但不在此版本

| 能力 | 计划版本 | 说明 |
|------|---------|------|
| 更多图形（五边形、八边形、圆柱等） | v1.1 | 正多边形可复用 `makePolygonPoints()` |
| 动画效果 | v1.2 | 绘制动画、图形过渡 |
| 自定义颜色名称 | v1.1 | "画一个天空蓝的圆" → 让用户自定义颜色映射 |
| 快捷键辅助 | v1.1 | Ctrl+Z 撤销、Delete 删除等 |

## 6. 技术决策记录

| 决策 | 选择 | 替代方案 | 理由 |
|------|------|---------|------|
| 解析方式 | 关键词匹配 | NLP/LLM | 零依赖、确定性结果、零延迟 |
| 同音字处理 | 关键词组扩展 | 全局归一化 | 避免单字替换副作用（如"花"→"画"破坏"花朵"） |
| 匹配策略 | 长词优先 | 按定义顺序 | 避免短词截胡（如"蓝色"被"蓝"先匹配） |
| 图形渲染 | 内联 SVG | Canvas API | 对象级操作、无损缩放、DOM 事件绑定 |
| 状态管理 | React useState | Redux/Zustand | 项目规模小，不需要额外状态库 |
| 构建工具 | Vite | Webpack/CRA | 开发体验好，HMR 快 |
